# CREATE_ISSUE.md — Procedura creazione issue

> Riferimento rapido. La procedura completa è nella skill `create-issue`.

## Procedura

1. **Raccolta minima** — repo, tipo (bug/feature/improvement), obiettivo
2. **Crea issue** su GitHub con template leggero
3. **Kanban → Backlog** (`./scripts/kanban-move.sh` o `gh project item-add`)
4. **Conferma a Davide** con link

Non approfondire — i dettagli arrivano con `/issue-validate`.
